How it ties together. A single TLDv meeting fans out: transcript → normalize → chunk + embed → KB row + classification, while entities (attendees, mentioned companies, topics) land in Neo4j. When an agent later asks "what should we do about this meeting?", loadContext() joins the RAG hit, the calendar event, every contact's prior thread history, and the agent's own memory — then dispatches to the right output channel: a Kanban work item, a contract draft, a blog + LinkedIn pair, or a briefing card. Postgres is the source of truth; Neo4j is the relationship index; the RAG layer is the semantic index. All three are queried in parallel at context-load time.
