_Findings that confirm the current spec approach. Reference specific spec sections._

- **Designing AI agents to resist prompt injection** — OpenAI's published defensive architecture for prompt injection resistance — constraining risky actions at the infrastructure layer rather than relying on model behavior — directly validates P2 and is the same design bet Optimus makes with constitutional gates G1-G7 and the Communication Gateway risk tiers. The 'sensitive data protection in agent workflows' framing maps precisely to Optimus's content sanitization requirement in §5. This is peer-reviewed operational confirmation that infrastructure-layer enforcement outperforms prompt-layer enforcement, strengthening the spec's foundational assumption. Validates §0 P1/P2 Design Principles, §5 Guardrail Enforcement, §6 Tool Registry, §8 Audit System.
- **The creator of Claude Code just revealed his workflow, and developers are losing their minds** — Boris Cherny's disclosure of running 5 parallel Claude agents with notification-based coordination — described as 'Starcraft-like fleet commanding' — directly validates Optimus's parallel multi-tier agent architecture. The notification-triggered handoff pattern maps closely to Optimus's pg_notify event bus and the orchestrator-to-executor task delegation model. This is a first-party Anthropic workflow endorsing the architectural direction Optimus has already committed to in §2–§3. Validates §2 Agent Tiers, §3 Task Graph, §4 Tool Registry.
- **Box Maze: A Process-Control Architecture for Reliable LLM Reasoning** — Box Maze proposes explicit architectural layers for reasoning process control — memory grounding, structured inference, boundary enforcement — and empirically demonstrates that architectural constraints reduce boundary failure rates from ~40% (RLHH baseline) to below 1% under adversarial conditions. This directly validates Optimus's P2 principle that infrastructure enforces while prompts advise, providing quantitative evidence for the design bet. The three-layer decomposition (memory, inference, boundary) maps cleanly onto Optimus's context-loader → agent → constitutional-engine pipeline and strengthens the case for investing in infrastructure-layer enforcement over prompt hardening. Validates §0 P2 Design Principles, §5 Guardrail Enforcement, §8 Audit System, §9 Constitutional Layer.
- **OWASP LLM Top 10: OWASP Top 10 for Large Language Model Applications - 2025** — The 2025 OWASP Top 10 for LLMs is the current canonical threat taxonomy for LLM-based systems, covering prompt injection, excessive agency, insecure plugin/tool design, and supply chain vulnerabilities. The 2025 edition explicitly expanded coverage for agentic AI threat vectors that map directly to Optimus's P1/P2 enforcement model and the tool sandboxing layer in §6. Serves as external validation of the threat categories Optimus is designed to address and should be used to audit current guardrail coverage against the full Top 10. Validates §0 P1/P2 Design Principles, §5 Guardrail Enforcement, §6 Tool Integrity, §8 Constitutional Governance.

While superseded by the 2025 edition, it established the core threat categories (prompt injection, training data poisoning, insecure output handling) that informed Optimus's constitutional gate design. Useful as historical baseline to track which threats have evolved or been added since. Validates §5 Guardrail Enforcement, §8 Constitutional Governance.
